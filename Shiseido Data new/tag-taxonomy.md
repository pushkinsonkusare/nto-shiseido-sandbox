# Shiseido Catalog Tag Taxonomy — DRAFT v0.1

**Purpose:** single source of truth for what each filter *means*. Tags are never
assigned by eyeballing a product — they're always *derived* from atomic Fact
columns in `shiseido-product-images-manifest.csv` via the rules below. When you
add a new category (moisturizer, serum, cleanser…), add new Facts + new Tag
rules here; never redefine an existing tag's meaning.

Two kinds of tags:
- **Universal** — meaning doesn't depend on product category. Apply to any row
  that has the underlying facts, sunscreen or not.
- **Category-scoped** — only make sense within one product type (e.g. sunscreen
  format claims). Prefixed for clarity (`sun:*`).

Regeneration is mechanical (a script, not an LLM): for each row, evaluate every
rule against that row's Fact columns; if true, append the tag to `Tags`.

---

## 1. Fact columns (new columns to add to the manifest CSV)

### 1a. Universal facts (apply to every category, fill in as each category is tagged)

| Column | Type | Values | Notes |
|---|---|---|---|
| `Fact: Fragrance Free` | bool | true/false | From ingredient list or explicit claim |
| `Fact: Sensitive Skin Claim` | bool | true/false | Explicit "for sensitive skin" / "dermatologist tested" language |
| `Fact: Time Of Day` | enum | AM / PM / Both | When product copy specifies |
| `Fact: Texture Finish` | enum | sheer / dewy / matte / rich / lightweight | From Overview/Key Benefits text |
| `Fact: Treatment Claim` | bool | true/false | Does it make an anti-aging/brightening/firming claim (vs. pure protection/cleansing)? |

### 1b. Sunscreen-specific facts

| Column | Type | Values | Notes |
|---|---|---|---|
| `Fact: Format` | enum | stick / lotion / cream / spray / powder | |
| `Fact: SPF Value` | integer | e.g. 40, 50, 60 | Parsed out of product name/copy so it's sortable/filterable numerically |
| `Fact: Filter Type` | enum | mineral / chemical | |
| `Fact: Water Resistant` | bool + duration | true/false, minutes if stated | |
| `Fact: Reapply Friendly Format` | bool | true/false | Stick/powder/spray that works over makeup |
| `Fact: Antioxidant Claim` | bool | true/false | Urban Environment's core pitch |

### 1c. Future categories (placeholders — fill in when tackled)

- Moisturizer: `Fact: Richness`, `Fact: SPF Value` (reuse from 1b if it's a moisturizer+SPF combo)
- Serum: `Fact: Active Ingredient Family` (retinol-alt, vitamin C, peptide…), `Fact: Booster vs Standalone`
- Cleanser: `Fact: Cleanser Base` (foam/oil/water/milk), `Fact: Makeup Removal Claim`

## 2. Tag rules

### 2a. Universal tags (reusable across every category)

| Tag | Rule |
|---|---|
| `fragrance-free` | `Fact: Fragrance Free` = true |
| `sensitive-skin-friendly` | `Fact: Fragrance Free` = true AND (`Fact: Filter Type` = mineral OR `Fact: Sensitive Skin Claim` = true) |
| `treatment-plus-protection` | `Fact: Treatment Claim` = true AND `Fact: SPF Value` is not null |
| `am-routine` / `pm-routine` | `Fact: Time Of Day` = AM / PM |

### 2b. Sunscreen tags (mirrors the 5 tiles in the screenshot)

| Tag | Rule | Sanity-check against real rows |
|---|---|---|
| `sun:active-outdoor` | `Fact: SPF Value` >= 50 AND `Fact: Water Resistant` = true | Protector Clear Stick SPF 60+, Protector Lotion Mineral SPF 60+, Protector Lotion SPF 60+ (all Ultimate Sun) |
| `sun:urban-everyday` | `Fact: SPF Value` between 30–50 AND `Fact: Antioxidant Claim` = true | Urban Environment Oil-Control/Vita-Clear/Fresh-Moisture SPF 40-42 |
| `sun:gentle-sensitive` | = `sensitive-skin-friendly` (universal tag) restricted to sunscreen rows | Protector Lotion Mineral SPF 60+, Urban Environment Mineral Clear SPF 50 |
| `sun:reapply-on-the-go` | `Fact: Reapply Friendly Format` = true | Protector Clear Stick SPF 60+ (both lines) |
| `sun:spf-with-skincare` | `Fact: Treatment Claim` = true AND `Fact: SPF Value` is not null AND Collection NOT IN (Ultimate Sun, Urban Environment) | Vital Perfection Advanced Day Cream SPF 30, Benefiance Wrinkle Smoothing Day Cream SPF 23, Future Solution LX Total Protective Cream SPF 30, Essential Energy Hydrating Day Cream SPF 20 |

Note `sun:spf-with-skincare` is really just `treatment-plus-protection` (2a) with
the two suncare-only lines excluded — kept as a separate row here so the mapping
to the screenshot's 5th tile is explicit, but it's built entirely from the
universal tag + a collection filter, not a new concept.

---

## 3. Computed column

`Tags` — semicolon-delimited list, e.g.:
```
fragrance-free; sensitive-skin-friendly; sun:gentle-sensitive
```
Regenerated by re-running the rule pass whenever a Fact column or a rule in
this file changes. Cursor/agent filters against this column directly at query
time — no re-reasoning needed per query.
